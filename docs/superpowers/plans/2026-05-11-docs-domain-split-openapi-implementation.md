# Docs Domain Split + OpenAPI REST Reference Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split oversized docs into domain-focused pages and integrate true OpenAPI-rendered REST API docs in Docusaurus.

**Architecture:** Keep handwritten conceptual docs in MDX, move environment variable content into a nested domain tree, and make OpenAPI spec the source-of-truth for REST endpoint reference pages generated by Docusaurus OpenAPI tooling. Preserve existing navigation clarity by adding explicit grouped sidebar categories and replacing manual REST endpoint content with a bridge page to generated reference routes.

**Tech Stack:** Docusaurus 3.10.1, MDX, TypeScript config (`docusaurus.config.ts`, `sidebars.ts`), OpenAPI 3.0 YAML

  - name: Collaboration
